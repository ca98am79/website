Welcome to the Coding Workshop

This group will serve as a gathering place for UMD library staff interested in computer programming. We will meet weekly to learn coding together, talk about code, and bounce ideas off each other. CodeAcademy's Python track will serve as a scaffolding for the workshop. Python is a popular language among hobbyists and real programmers alike for it's [what?].

Half of each weekly meeting will be devoted to working through CodeAcademy lessons together. Working through the exercises as a group will allow participants to ask others for help or talk about ideas that come up. In addition, the structure will hopefully serve as a motivating factor.

Why are we doing this?

See the links for better explanations but basically, coding is a good skill to have for three reasons:

  1. Given that so many library resources and tools are digital now, programming allows us to take a more active role in managing information resources, rather than simply being passive consumers. Coding allows you to build basic, artisanal tools that are useful to a small community, or even just you. A basic application that maybe doesn't warrant full-on IT support can sometimes be built "out of the back of the garage" to make your own job easier.
  2. A basic knowledge of code allows better communication with IT departments and vendors.
  3. Many library tools (Libguides, SFX, WorldcatUMD) can be customized, extended, or fixed by employing a bit of code.
